# Supply Chain PO Analytics Dashboard

## Project Overview
This project analyses purchase order data to improve visibility across supplier performance, open purchase orders, delayed shipments and inventory risks.

## Business Problem
Supply chain teams need accurate and timely reporting to understand which purchase orders are open, which suppliers have delays, which confirmations are pending, and where follow-up action is required.

## Tools Used
- Excel
- SQL
- Power BI

## Key KPIs
- Total PO Value
- Open PO Count
- Delayed PO Count
- Pending Supplier Confirmations
- Average Delay Days
- PO Value by Supplier
- PO Count by Status

## Analysis Performed
- Cleaned and validated purchase order data
- Calculated PO Value using Order Quantity and Unit Cost
- Calculated Remaining Quantity
- Identified delayed purchase orders
- Checked pending supplier confirmations
- Summarised supplier performance using pivot tables
- Designed a Power BI dashboard for operational visibility

## Key Insights
- High-value open purchase orders should be monitored closely.
- Delayed shipments may affect inventory availability and customer commitments.
- Pending supplier confirmations require regular follow-up.
- Supplier-level reporting helps operations teams prioritise action.

## Business Recommendations
- Prioritise delayed high-value purchase orders.
- Follow up pending supplier confirmations weekly.
- Monitor suppliers with repeated ETA changes.
- Improve reporting accuracy by validating PO value, remaining quantity and date fields.
